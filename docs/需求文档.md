## 需求文档

最小可行性的B2C在线商城系统

功能：商品展示 购物车 订单流程 在线支付 用户注册登录 基础后台管理

角色：

1. 游客：浏览商品
2. 注册用户：购买商品，管理自己的订单和收货地址
3. 管理员：管理全部用户，订单和商品

状态

- 角色状态：未创建 已创建 已禁用

- 商品状态： 创建 上架 下架 售罄

- 订单状态：待付款 待发货 已关闭 待发货 已完成 退款中 已退款 退款失败

功能模块

- 用户：注册登录

- 商品：商品列表 详情

- 订单：订单确认 提交订单 订单列表 订单详情 订单操作

- 支付：发起支付 支付回调

流程

- 用户订单处理流程

- 后台订单处理流程

非功能需求：

1. 性能：加载页面时间
2. 安全：密码加密 支付接口HTTPS 防止SQL注入和XSS
3. 兼容：浏览器

数据统计（后台）

- 数据看板
